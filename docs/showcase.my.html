ရိုးရှင်းသော စကားလုံးဖြင့် ပြောပါ။ studio အဆင့် content ရယူပါ။

ဗီဒီယို၊ စာတန်းထိုး၊ အသံသွင်း၊ ရုပ်ပုံ နှင့် ဂီတ — စီစဉ်၊ စျေးနှုန်းသတ်မှတ်၊ ထုတ်လုပ်ပြီး တင်ရန်အသင့် ပြင်ဆင်ပေးသည်။ လူတစ်ယောက်နှင့် ပြောသကဲ့သို့ ပြောဆိုပါ။ နောက်ကွယ်တွင် စည်းကမ်းရှိ၊ ထပ်ခါတလဲလဲ လုပ်နိုင်သော ထုတ်လုပ်မှုလုပ်ငန်းစဉ်ကို လည်ပတ်ပေးသည်။

“ကျွန်တော့် ကော်ဖီ brand အတွက် TikTok ကြော်ငြာ လုပ်ပေးပါ။” “မြန်မာဘာသာဖြင့် စာတန်းထိုး ထည့်ပေးပါ။” “နွေးထွေးသော အသံသွင်း တစ်ခု ဖန်တီးပေးပါ။” “claymation ရှင်းလင်းချက်ဗီဒီယို အဖြစ် ပြောင်းပေးပါ။”
ကျွမ်းကျင်သော လုပ်ငန်းစဉ်

လုပ်ငန်းတိုင်း တူညီသော လေးဆင့်ဖြင့် လည်ပတ်သည်

ခန့်မှန်းစရာမလို၊ မမျှော်လင့်ထားသော ငွေတောင်းခံလွှာမရှိ၊ “မြင်ကွင်းတိုင်း လူသစ်တစ်ယောက်၏ မျက်နှာ” မရှိ။ ဤအခြေခံသည် အောက်ပါအရာအားလုံး၏ အောက်တွင် တည်ရှိသည် — ၎င်းသည် output ကို တစ်ညီတည်းဖြစ်စေပြီး client ထံ လက်ဆင့်ကမ်းရန် ဘေးကင်းစေသည်။

01

Brand ကို သတ်မှတ်ပါ

သင့်အမည်၊ ပရိသတ်၊ အရောင်များ၊ လေသံ နှင့် ဘယ်တော့မှ မပြောစေချင်သောအရာများကို တစ်ကြိမ်တည်း မှတ်သားပြီး နေရာတိုင်းတွင် ပြန်လည်အသုံးပြုသည်။

02

ကုန်ကျစရိတ် သဘောတူပါ

မည်သည့်အရာမျှ မဖန်တီးမီ ခန့်မှန်းငွေကို ကြိုတင် မြင်ရသည်။ budget အမြင့်ဆုံးကို code ဖြင့် ထိန်းချုပ်ထားသည် — အလွန်အကျွံ သုံးစွဲ၍မရ။

03

အစမ်းကြည့်၊ ပြီးမှ ထုတ်လုပ်

စျေးသက်သာသော အစမ်းများ ဦးစွာ။ သင် ဦးတည်ချက်ကို ရွေးချယ်သည်။ ထို့နောက်မှသာ ပြီးဆုံးသော render အတွက် ငွေသုံးစွဲသည်။

04

တင်ရန် Package ပြင်ဆင်

platform တစ်ခုစီအလိုက် အရွယ်အစားမှန်ကန်စွာ၊ စာတန်းထိုး၊ hashtag နှင့် alt text များနှင့်အတူ — ၎င်းကို မည်သို့ ဖန်တီးခဲ့ကြောင်း အတိအကျ မှတ်တမ်းနှင့်အတူ။

တကယ့်ပရောဂျက်များ · အဆင့်လိုက်

၎င်းဖြင့် တကယ်ဖန်တီးထားသော ရုပ်ရှင်နှစ်ကား

နမူနာများ မဟုတ် — တကယ့် pipeline၊ တကယ့် output အရေအတွက်၊ ပရောဂျက် ledger မှ တကယ့်ကုန်ကျစရိတ်ဖြင့် အစမှအဆုံး ထုတ်လုပ်ထားသော တကယ့်ရုပ်ရှင်တိုများ။ နှစ်ကားစလုံးသည် ~၉၂ စက္ကန့်ကြာ papercraft ရုပ်ရှင်တိုများ ဖြစ်သည်။

Case study · Papercraft ဂုဏ်ပြုချက်

Emberfell II: The Gates

3D papercraft 16:9 · YouTube ၁၀၄ စက္ကန့် · ၁၂ ခန်း
Use case ဇာတ်လမ်းအကြောင်း agent နှင့် စကားပြောရုံပါပဲ။ prompt မရေး၊ shot list မလို — Arthas နှင့် Culling of Stratholme အကြောင်း စကားဝိုင်းတစ်ခုကနေ ဇာတ်ညွှန်းရေး၊ ဇာတ်ကောင်ချ၊ shot တစ်ခုစီအတွက် model ရွေး၊ ပြီးတော့ ရိုက်ကူးပေးခဲ့သည်။
မည်သို့ ဖန်တီးခဲ့သည် ကြည့်ရန်အသေးစိတ်ကို ဖျောက်ရန်

ရည်ရွယ်ချက်: Warcraft III: Reign of Chaos ၏ အခက်ခဲဆုံး မြင်ကွင်းဖြစ်သော Culling of Stratholme — ရောဂါကူးစက်ခံထားရသော မြို့တစ်မြို့ကို လူသားဘဝ မပျက်သေးခင် Arthas က သတ်ဖြတ်ရန် အမိန့်ပေးသည့် အခန်း — ကို ဂုဏ်ပြုသည့် ~၁၀၅ စက္ကန့်ကြာ papercraft တစ်ကား။ မူရင်း ဇာတ်ကောင်များ နှင့် မူရင်း စကားပြောများဖြင့် ဖြတ်စက္ကူအဖြစ် ပြန်လည်စိတ်ကူးထားသည်: မိမိ၏ ပြည်သူများအပေါ် တံခါးပိတ်ရမည့် မင်းသား၊ မလုပ်ပါနှင့်ဟု တောင်းပန်သော ဆရာ၊ နှင့် ထွက်ခွာသွားသော ဆေးဆရာမ။ အခြေအနေကို ကိုးကားသည်၊ ကူးယူချက် မဟုတ်ပါ။

သင် agent ကို ဤသို့ ပြောခဲ့သည် “Warcraft III မှ Arthas အကြောင်း ပြောပြပါ — ဘယ်ဇာတ်လမ်းက အာရုံစိုက်မှု ရနိုင်မလဲ? papercraft ကား အဖြစ် လုပ်ပါ၊ ၂ မိနစ်အောက်။ သရုပ်ဆောင်ခန်းများအတွက် Omni Flash၊ တိတ်ဆိတ်သော ခန်းများအတွက် Veo 3.1 lite သုံးပါ။”
ပွင့်လင်းမြင်သာမှု — အမှန်တကယ် ထုတ်လုပ်ခဲ့သော ဇာတ်ကောင် model sheet များ
ဇာတ်ကောင်တစ်ဦးလျှင် ပုံတစ်ပုံ — ရှုထောင့်တစ်ခုလျှင် ပုံတစ်ပုံ မဟုတ်ပါ — ထို့ကြောင့် ဇာတ်ကောင်တစ်ဦးလုံး၏ အထောက်အထားသည် ကိုးကားနေရာ တစ်ခုတည်းဖြင့် သယ်ဆောင်နိုင်သည်။ ဇာတ်ကောင်ပါဝင်သော keyframe တိုင်းက ဤ sheet များကို ကိုးကားသည် (Corin ၏ sheet ကို ၇ ခုတွင်) — ထို့ကြောင့် မျက်နှာများ တစ်ကားလုံး တည်တံ့သည်။
မည်သို့ ဖန်တီးခဲ့သည် — တကယ့် pipeline
  1. ဇာတ်ကောင်များ ပုံသေချုပ် — Corin ကို ပထမကားမှ ဆက်လက်သုံးစွဲ၊ Veyron နှင့် Sera ကို စာသားမှ model sheet အသစ်များအဖြစ် ဖန်တီးပြီး registry သို့ သိမ်း။
  2. keyframe ၁၂ ခု ဖန်တီး — မြင်ကွင်းတစ်ခုစီအတွက် အတည်ပြုပြီးသား ရုပ်ပုံငြိမ်၊ တစ်ခုစီသည် ပုံသေချုပ်ထားသော sheet များကို ကိုးကားသည်။ မည်သည့်ဗီဒီယိုအတွက်မှ ငွေမပေးမီ အတည်ပြုပြီး။
  3. shot အလိုက် model ရွေးချယ် — သရုပ်ဆောင်ခန်း ၈ ခုကို Omni 3d-papercraft (မျက်နှာ၊ စကားပြော၊ နှုတ်ခမ်းလှုပ်ရှားမှု)၊ တိတ်ဆိတ်သောခန်း ၄ ခုကို Veo 3.1 lite ($0.03/s) — ပြာကျခြင်း၊ လည်နေသော pinwheel၊ တိတ်ဆိတ်သော နောက်ဆက်တွဲ။
  4. clip များကိုပင် စကားပြောစေ — Omni သည် စကားပြော နှင့် အသံကို တစ်ကြိမ်တည်း ထုတ်ပေးသည်။ သီးခြား အသံသွင်း မလို။ ၁၀၆ စက္ကန့် ဂီတသည် အောက်တွင် ရှိသည်။
  5. ပေါင်းစပ် — မြင်ကွင်း ၁၂ ခုကို မူရင်းအသံ အပြည့်အဝ နှင့် ဂီတ ၁၄% ဖြင့် ချုပ်ဆက်။

A Second Either Way

3D papercraft 16:9 · YouTube ၉၂ စက္ကန့် · ၁၁ ခန်း
Use case YouTube link တစ်ခု ထည့်ပြီး စတိုင်ကို ပြောလိုက်ရုံပါပဲ။ “ဤဗီဒီယိုကို ခွဲခြမ်းစိတ်ဖြာပြီး papercraft စတိုင်ဖြင့် ဆင်တူတစ်ခု လုပ်ပေးပါ၊ ၂ မိနစ်အောက်။” ၅:၁၈ မူရင်းကို ကြည့်ပြီး scene ၇၀ အဖြစ် ခွဲခြမ်း၊ မူရင်းဇာတ်ကောင်များဖြင့် ဖွဲ့စည်းပုံကို ပြန်တည်ဆောက်ခဲ့သည်။
မည်သို့ ဖန်တီးခဲ့သည် ကြည့်ရန်အသေးစိတ်ကို ဖျောက်ရန်

ရည်ရွယ်ချက်: ရှိပြီးသား ရုပ်ရှင်တိုတစ်ကား (“ONE SECOND LATE”) ၏ ဖွဲ့စည်းပုံနှင့် အရှိန်အဟုန် ကို papercraft “sliding-doors” အချစ်ဇာတ်လမ်းအဖြစ် ပြန်လည်ဖန်တီးရန် — လုံးဝမူရင်းဇာတ်ကောင်များနှင့် စကားပြောကို စကားရပ်ပြောင်းလဲ၍၊ ရုပ်ရှင်ရိုက်ကွက် သို့မဟုတ် ပုံသဏ္ဌာန်များ ကူးယူခြင်းမရှိဘဲ၊ 5:18 မှ ၂ မိနစ်အောက်သို့ ချုံ့ခြင်း။

သင် agent ကို ဤသို့ ပြောခဲ့သည် “[YouTube link တစ်ခု] — ဤဗီဒီယိုကို ခွဲခြမ်းစိတ်ဖြာပါ၊ ထိုကဲ့သို့ ဒါပေမဲ့ papercraft စတိုင် ဖြင့် တစ်ခု ဖန်တီးချင်တယ်။ ၂ မိနစ်ထက် မကျော်ဖို့။”
ပွင့်လင်းမြင်သာမှု — အမှန်တကယ် ထုတ်လုပ်ခဲ့သော ဇာတ်ကောင် character sheet များ (အဆင့် ၂ ရလဒ်)
Nora ၏ ခြယ်လှယ်ပြန်ထားသော အနီရောင် scarf ကို character sheet ထဲမှာပင် သတ်မှတ်ထားသောကြောင့် scene တိုင်းတွင် ထပ်မဖော်ပြဘဲ တည်မြဲနေသည်။

  1. ကိုးကားဗီဒီယိုကို ခွဲခြမ်းစိတ်ဖြာ — agent သည် 5:18 မူရင်းကို ကြည့်ရှုပြီး ၎င်း၏ ဖွဲ့စည်းပုံ၊ အရှိန်အဟုန် နှင့် စတိုင်ကို မြင်ကွင်း ၇၀ ခွဲ၍ အနှစ်ချုပ်ကာ ပြန်လည်ဖန်တီးရေးအစီအစဉ်ကို ထုတ်ပေးသည်။ (~$0.17)
  2. မူရင်းဇာတ်ကောင် ၂ ဦး ပုံသေချုပ် — Eli နှင့် သူမ၏ အမှတ်အသား နီရောင်ပုဝါနှင့် Nora (မူရင်းရုပ်ရှင်၏ တွေ့ဆုံ prop ကို ပြန်လည်တီထွင်ထားခြင်း) — character sheet များကို registry သို့။
  3. keyframe ၁၁ ခု ဖန်တီး — ပုံသေချုပ်ထားသော ဇာတ်ကောင်များကို ကိုးကားသည့် အတည်ပြုပြီး ရုပ်ပုံငြိမ်များ။
  4. clip ၁၁ ခုသို့ လှုပ်ရှားစေ — Omni 3d-papercraft၊ image-to-video၊ clip အလိုက် camera move (တစ်ခုစီ ၈–၁၀ စက္ကန့်)။
  5. အသံ & ဂီတ — ပြီးမှ ဆုံးဖြတ်ချက်တစ်ခု — အသံ ၃ မျိုးဖြင့် voiceover ထုတ်ခဲ့သော်လည်း ပြန်နားထောင်ကြည့်ရာ clip များ၏ မူရင်း Omni အသံက ပိုကောင်းသဖြင့် TTS ကို ဖယ်ပြီး မူရင်းအသံကို ထားခဲ့သည်။
  6. ပေါင်းစပ် — ပြိုင်တူအချိန်ဇယား ဖွဲ့စည်းပုံ မပျက်ဘဲ ၉၂ စက္ကန့်ကြာ နောက်ဆုံးဖြတ်ချုပ်အဖြစ် ချုပ်ဆက်။

Omni ဗီဒီယို

Gemini · core ~$1.03 / 10-second clip

သင်ရရှိသည့်အရာ: လှုပ်ရှားမှု နှင့် အသံ အတူတကွ ဖန်တီးထားသော တို၍ ပြီးဆုံးသည့် ဗီဒီယို clip တစ်ခု (~၁၀ စက္ကန့်အထိ) — ကုန်ပစ္စည်း sizzle၊ စကားပြော ရှင်းလင်းချက် သို့မဟုတ် ရွေးချယ်ထားသော အနုပညာစတိုင်ဖြင့် မြင်ကွင်းတစ်ခု။ ၎င်းသည် သင့်တွင် ရှိပြီးသား clip တစ်ခုကိုလည်း တည်းဖြတ်နိုင်သည်: မျက်နှာပြင်ပေါ်စာသား၊ အသံ effect များ ထည့်ခြင်း၊ သို့မဟုတ် အစမှ ပြန်မဖန်တီးဘဲ စတိုင်ပြောင်းခြင်း။

သင် agent ကို ဤသို့ ပြောသည် “ကျွန်တော့် ကော်ဖီအိတ်ကို ၁၀ စက္ကန့်ကြာ claymation ရှင်းလင်းချက် လုပ်ပေးပါ၊ နွေးထွေးသော မနက်ခင်းအလင်း၊ ဖြည်းဖြည်း push-in။”
သင့် studio က မည်သို့ ဖန်တီးသည်
  1. အစပြုနေရာ ရွေးပါ — စာသား prompt၊ keyframe ရုပ်ပုံတစ်ပုံ၊ ကိုးကားရုပ်ပုံ ၅ ပုံအထိ၊ သို့မဟုတ် တည်းဖြတ်ရန် ရှိပြီးသား ဗီဒီယိုတစ်ခု။
  2. အနုပညာစတိုင် တစ်ခု ရွေးပါ (သို့မဟုတ် photoreal ထားပါ)။ ၎င်းသည် အမြဲ သင့်ဆုံးဖြတ်ချက်ဖြစ်သည် — တိတ်တဆိတ် ဘယ်တော့မှ မရွေးပါ။
  3. စမ်းသပ်ပြီးသား preset ၄၆ ခုမှ camera move တစ်ခု အဆိုပြုပါ — clip တစ်ခုလျှင် သန့်ရှင်းသော move တစ်ခု။
  4. ကုန်ကျစရိတ်ကို ခန့်မှန်း၊ အတည်ပြုပြီး render လုပ်ပါ။ တိကျသော clip ကို ပြန်ဖန်တီးနိုင်ရန် prompt ကို သိမ်းဆည်းထားသည်။

စာသားကို နေရာတစ်ခုတည်းတွင် အတည်တကျ ချထားပြီး အဆင်ပြေမည်ဟု မျှော်လင့်ရုံ မဟုတ်ပါ။ render မလုပ်မီ agent သည် ဗီဒီယို frame အစစ်များနှင့် အသံကို စစ်ဆေးပြီးမှ စကားလုံးများ မြင်ကွင်းနှင့် ကိုက်ညီစေရန် — ဖတ်ရလွယ်ကူ၊ ဘရန်းနှင့် လိုက်ဖက်ကာ မျက်နှာကို ဘယ်တော့မှ မဖုံးအောင် — အသွင်အပြင် တိုင်းကို ဆုံးဖြတ်ပေးသည်။

နေရာလွတ် ဘယ်မှာရှိသလဲ၊ နောက်ခံသည် တောက်ပ၊ မှောင်မိုက် သို့မဟုတ် ရှုပ်ထွေးနေသလား၊ ဇာတ်ဆောင်၏ မျက်နှာ ဘယ်နေရာတွင် ရှိသလဲ — frame များကို ခွဲခြမ်းစိတ်ဖြာ၍ ရှာဖွေသည်။

အသံကို ဖတ်သည်

စာသားမှ စကားလုံးတစ်လုံးချင်းစီ၏ တိကျသော အချိန်ကိုက်များနှင့် ဘယ်စကားလုံးများက အလေးပေးမှုကို သယ်ဆောင်သည်ကို သိရှိ၍ — စာတန်းများ ရစ်သမ်နှင့်အတူ ကျရောက်စေသည်။

ထို့နောက် — မြင်ကွင်းအလိုက် ဆုံးဖြတ်သည်

နေရာ

အလွတ်ဆုံး နေရာ (ကောင်းကင်၊ နံရံ၊ ရေ) တွင် ချထားသည် — မျက်စိမှိတ်ကာ အလယ်တွင် မဟုတ် — မျက်နှာနှင့် platform ၏ ခလုတ်နေရာမှ ကင်းလွတ်စေသည်။

အရွယ်အစား

hook စကားလုံး ကို ကြီးကြီးထားသည် (frame အနားမှ ကျော်ထွက်နိုင်သည်); စာသားစာကြောင်းက ရိုးရိုးသာ ဖြစ်သည်။ အရွယ်အစားသည် စကားလုံး၏ အရေးပါမှုကို လိုက်၍ ပြောင်းသည်။

အရောင်

နောက်ခံနှင့် ချိန်ဆ၍ — မှောင်ပေါ်တွင် အလင်း၊ လင်းပေါ်တွင် အမှောင် — အဓိက စကားလုံး ၁–၂ လုံးကို သင့်ဘရန်း accent သို့ ပြောင်းလဲပေးသည်။

ဖတ်ရလွယ်ကူမှု

ရှုပ်ထွေး သို့မဟုတ် အသားအရေ ကြမ်းသော နောက်ခံပေါ်တွင် စာသား မပျောက်ပျက်စေရန် ၎င်း၏ နောက်တွင် မှုန်ဝါးသော scrim သို့မဟုတ် pill ကို ထည့်ပေးသည်။

စတိုင်

မြင်ကွင်းအလိုက် ရွေးချယ်သည် — pill စာသား၊ hero punch-in၊ အထက်ပိုင်း တည်နေရာ တံဆိပ်၊ သို့မဟုတ် အထင်ကရ ဇာတ်ဆောင်နောက်ကွယ်မှ စာသား။

အက်နီမေးရှင်း

spring ဝင်ရောက်မှု ဖြင့် တစ်ကြိမ်လျှင် စာကြောင်းတစ်ကြောင်းစီ၊ အသံ ရစ်သမ်နှင့် အချိန်ကိုက် — ဖတ်ရလွယ်ကူ၍ စာသားတန်း တစ်ခုလုံး ဘယ်တော့မှ မဖြစ်စေပါ။

စွမ်းဆောင်ရည်

အသံသွင်း

လမ်းကြောင်း နှစ်မျိုး: အခမဲ့ ရိုးရှင်းသော narrator တစ်ခု၊ သို့မဟုတ် စာကြောင်းလိုက် ချိန်ညှိသည့် ခံစားမှုပါ၊ ညွှန်ကြားထားသော အသံတစ်ခု။

ပရောဂျက်တိုင်းတွင် budget အမြင့်ဆုံးနှင့် သုံးစွဲမှု ledger ရှိသည်။ ခန့်မှန်းငွေ ဦးစွာ ပြသသည်။ အမြင့်ဆုံးကို code ဖြင့် ထိန်းချုပ်ထားသဖြင့် အလွန်အကျွံ ရုပ်ပိုင်းအရ သုံးစွဲ၍မရ။

တစ်ညီတည်းဖြစ်မှုကို အင်ဂျင်နီယာဆန်ဆန် တည်ဆောက်

ဇာတ်ကောင်များ၊ ကုန်ပစ္စည်းများ၊ logo များ နှင့် အသံများကို တစ်ကြိမ်တည်း ဖန်တီး၍ ပုံသေချုပ်ထားသည်။ တူညီသော မျက်နှာနှင့် အသံသည် campaign တစ်ခုလုံးတွင် သယ်ဆောင်သည်။

သက်သေအထောက်အထား တကယ်ဖြစ်နေစေ

studio ကို ချီးမွမ်းစကား၊ စာရင်းအင်း သို့မဟုတ် သတင်းများ တီထွင်ခြင်းမှ တင်းကျပ်စွာ တားမြစ်ထားသည်။ သင့်ကိန်းဂဏန်းများ တစ်လုံးမကျန် — သို့မဟုတ် ဘာမှ မရှိ။ လုပ်ငန်းတိုင်း ၎င်းကို မည်သို့ ဖန်တီးခဲ့ကြောင်း အတိအကျ မှတ်တမ်း ထားရှိသည်။
