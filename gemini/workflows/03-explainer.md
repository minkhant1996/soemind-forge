# Explainer & Educational Videos

> Clear, engaging content that teaches concepts and demonstrates processes.

---

## When to Use

- Product tutorials
- Feature walkthroughs
- How-to guides
- Onboarding videos
- Educational content
- Documentation videos

**Ideal Duration**: 60-120 seconds (8-15 clips)

---

## Explainer Structures

### The Problem-Solution-Benefit (60s)

Best for: Product explainers, feature highlights

```
PROBLEM (0-15s)
├── Identify pain point
├── Show consequences
└── Build empathy

SOLUTION (15-45s)
├── Introduce product/feature
├── Show how it works
├── Step-by-step demo
└── Key moments highlighted

BENEFIT (45-60s)
├── Results achieved
├── Time/money saved
├── Call to action
```

---

### The What-Why-How (90s)

Best for: Educational content, concept explanations

```
WHAT (0-20s)
├── Define the concept
├── Context and relevance
└── "Here's what we're talking about"

WHY (20-45s)
├── Why it matters
├── Benefits of understanding
├── Real-world importance
└── "Here's why you should care"

HOW (45-80s)
├── Step-by-step breakdown
├── Visual demonstration
├── Key tips and tricks
└── "Here's how to do it"

RECAP (80-90s)
├── Summary of key points
├── Next steps
└── Call to action
```

---

### The Tutorial Flow (120s)

Best for: Step-by-step guides, how-to content

```
INTRO (0-10s)
├── What we'll learn
├── End result preview
└── Hook

OVERVIEW (10-20s)
├── High-level steps
├── What you'll need
└── Time estimate

STEP 1 (20-40s)
├── First action
├── Visual demonstration
└── Tips/warnings

STEP 2 (40-60s)
├── Second action
├── Visual demonstration
└── Common mistakes

STEP 3 (60-80s)
├── Third action
├── Visual demonstration
└── Pro tips

STEP N... (continue as needed)

RESULT (100-110s)
├── Show completed outcome
├── Before/after comparison
└── Success moment

OUTRO (110-120s)
├── Recap key steps
├── Related topics
└── Call to action
```

## Visual Styles for Explainers

### Screen Recording Style
```
Clean screen capture with cursor movements.
Highlighted click areas with subtle glow.
Zoom into important elements.
Consistent transition animations.
```

**Prompt for UI visualization**:
```
Clean mockup of a modern web application interface.
Dashboard with data visualization charts in blue and green.
Cursor clicking on a button, subtle ripple effect.
Soft drop shadow, white background, minimal design.
Screen recording aesthetic, professional SaaS quality.
```

### Animated Concept Style
```
Abstract visualization of concepts.
Icons and symbols representing ideas.
Smooth morphing transitions.
Consistent color palette.
```

**Prompt for concept visualization**:
```
Abstract 3D visualization of data flowing through a network.
Glowing blue nodes connected by light streams.
Information packets traveling along paths.
Dark background, clean minimal style.
Motion graphics aesthetic, corporate explainer quality.
```

### Live Action + Graphics Style
```
Real person demonstrating on screen.
Graphics overlay highlighting key points.
Split screen showing process and result.
Professional but approachable.
```

## Prompt Templates for Explainer Scenes

### The "Before" State
```
[SHOT TYPE] showing [USER TYPE] struggling with [PROBLEM].
[ENVIRONMENT] cluttered/confusing/frustrating.
[VISUAL ELEMENTS] showing complexity (multiple windows, papers, errors).
[LIGHTING] harsh or flat, emphasizing difficulty.
Camera [MOVEMENT] showing scope of problem.
Documentary style, relatable, authentic quality.
```

### The "Process" State
```
Clean [SHOT TYPE] of [INTERFACE/PRODUCT] in action.
[SPECIFIC FEATURE] highlighted with [VISUAL INDICATOR].
Step-by-step progression visible.
[USER] engaging with focused attention.
Soft, even lighting, clean environment.
Tutorial aesthetic, clear and professional.
```

### The "After" State
```
[SHOT TYPE] showing [USER TYPE] with [POSITIVE OUTCOME].
[ENVIRONMENT] organized/clear/efficient.
[VISUAL ELEMENTS] showing success (checkmarks, clean data, results).
Warm, balanced lighting, satisfying atmosphere.
Camera [MOVEMENT] revealing the improvement.
Aspirational but achievable, professional quality.
```

## Pacing for Explainers

| Content Type | Pace | Clip Duration |
|--------------|------|---------------|
| Intro/Hook | Fast | 3-5s |
| Concept explanation | Medium | 8-10s |
| Step demonstration | Slow | 10-15s |
| Tip/Warning | Medium | 5-8s |
| Result/Success | Slow | 8-10s |
| CTA | Fast | 3-5s |

## Audio for Explainers

| Tone | Use For | Voice Characteristics |
|------|---------|----------------------|
| Professional | B2B, enterprise | Clear, measured, authoritative |
| Friendly | Consumer, lifestyle | Warm, conversational, approachable |
| Energetic | Startup, tech | Upbeat, enthusiastic, modern |
| Calm | Wellness, finance | Soothing, trustworthy, reassuring |

### Music Style

| Phase | Music Type |
|-------|------------|
| Problem | Tense, minimal |
| Solution | Building, hopeful |
| Demo | Light, rhythmic |
| Success | Uplifting, resolved |

## Template: 60-Second Product Explainer

### Shot List

```
SCENE 1 (8s) - HOOK + PROBLEM
Visual: User overwhelmed by [OLD WAY]
Audio: "Tired of [PAIN POINT]?"

SCENE 2 (7s) - INTRODUCE SOLUTION
Visual: Product logo/interface reveal
Audio: "Meet [PRODUCT NAME]"

SCENE 3 (10s) - FEATURE 1
Visual: Demonstration of first key feature
Audio: "Simply [ACTION] to [BENEFIT]"

SCENE 4 (10s) - FEATURE 2
Visual: Demonstration of second key feature
Audio: "[FEATURE] automatically [DOES THING]"

SCENE 5 (10s) - FEATURE 3
Visual: Demonstration of third key feature
Audio: "Plus, [BONUS BENEFIT]"

SCENE 6 (8s) - RESULTS
Visual: Before/after or success metrics
Audio: "[USERS] have [ACHIEVED RESULT]"

SCENE 7 (7s) - CTA
Visual: Logo + CTA text
Audio: "Start your free trial at [URL]"
```

## Example: Feature Walkthrough

**Context**: Explaining an AI code assistant feature

**Scene 1: The Problem**
```
Over-shoulder shot of developer staring at screen full of complex code.
Multiple error messages in red, console showing failures.
Developer sighs, rubs temples in frustration.
Harsh blue monitor light, dark room, stressful atmosphere.
Camera slowly pushes in on the overwhelming screen.
Documentary style, relatable developer experience.
```

**Scene 2: The Solution**
```
Same developer's screen now showing clean IDE with AI assistant panel.
Soft glow from AI interface, purple and blue accent colors.
Developer's posture noticeably more relaxed.
Balanced lighting, organized workspace.
Camera reveals the new tool smoothly.
Product demo aesthetic, professional quality.
```

**Scene 3: Feature Demo**
```
Close-up of screen showing code editor.
Developer types a comment describing a function.
AI suggestions appear inline with subtle animation.
Green highlighting on suggested code.
Cursor moves to accept suggestion.
Screen recording style, clear and instructional.
```

**Scene 4: Success**
```
Medium shot of developer leaning back with satisfied smile.
Screen shows "All tests passing" with green checkmarks.
Warm afternoon light through window.
Clean desk, coffee cup, plant in background.
Relaxed, accomplished atmosphere.
Lifestyle commercial quality, aspirational.
```

- [ ] Steps logically ordered
- [ ] Each step has clear visual
- [ ] UI/screens mocked up or described
- [ ] Pacing appropriate for content
- [ ] Technical accuracy verified
- [ ] CTA is clear and actionable
- [ ] Voiceover script written
