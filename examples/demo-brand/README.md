# Demo Project: Sunrise Roasters ☕

A **complete, filled-in example project** for a fictional small-batch coffee
roastery. Use it two ways:

1. **Learn by reading** — see what a finished `project.md`, `brand.md`, and
   asset registry look like (the blank templates live in [`templates/`](../../templates/)).
2. **Try the kit right now** — copy it into `projects/` and start generating:

```bash
mkdir -p projects
cp -r examples/demo-brand projects/demo-brand
```

Then tell your AI agent:

```
Read AGENT-GUIDE.md first. Using the demo-brand project, create an Instagram feed
image announcing our new Ethiopia single-origin roast.
```

## What's inside

```
demo-brand/
├── README.md                 # This file
├── templates/
│   ├── project.md            # Filled project brief (campaign, audience, offer)
│   └── brand.md              # Filled brand guidelines (colors, tone, style)
└── config/
    └── assets.yaml           # Asset registry (characters, product, voice…)
```

## Making it yours

When you're ready for a real project, either:

- **Ask the agent to interview you:** run `/onboard-brand` (or say
  *"onboard my brand"*) and the agent fills these files from a conversation, or
- **Copy the blank templates** from [`templates/`](../../templates/) into
  `projects/<your-name>/templates/` and fill them by hand.

> Everything in this demo — the brand, people, stats, and testimonials — is
> **fictional**, invented to demonstrate the format. Never fake social proof
> for a real brand; the templates mark which fields must be real.
